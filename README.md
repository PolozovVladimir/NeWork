# NeWork - Social Network Android App

## Описание проекта

NeWork - это мобильное приложение социальной сети для Android, 
разработанное на Kotlin с использованием современных технологий и 
архитектурных паттернов.

## Основной функционал

###  Авторизация
- Регистрация новых пользователей
- Вход в систему
- Валидация полей ввода
- Обработка ошибок авторизации

###  Основные экраны
- **Посты** - просмотр, создание, лайки
- **События** - просмотр, создание, участие
- **Пользователи** - список пользователей, профили

### ️ Создание контента
- **Посты** с поддержкой:
  - Текстового контента
  - Изображений (загрузка на сервер)
  - Ссылок
  - Геолокации
  - Упоминаний пользователей
- **События** с поддержкой:
  - Описания события
  - Даты и времени проведения
  - Типа события (онлайн/офлайн)
  - Местоположения
  - Спикеров
  - Медиафайлов

###  Профили пользователей
- Просмотр профиля
- Стена пользователя (посты)
- История работы
- Управление собственным профилем

## Интеграция с картами
- Выбор местоположения на карте
- Отображение координат в постах и событиях
- Интерактивные карты Google Maps

###  Управление медиафайлами
- Загрузка изображений на сервер`
`- Валидация размера файлов (макс. 15MB)
- Предварительный просмотр изображений

## Технологический стек

###  Архитектура
- **Single Activity** - вся навигация через фрагменты
- **MVVM** - Model-View-ViewModel паттерн
- **Repository Pattern** - абстракция данных
- **Dependency Injection** - Dagger Hilt

###  Основные библиотеки
- **Kotlin** 1.9.24 - основной язык разработки
- **Kotlinx Coroutines** 1.9.0 - асинхронное программирование
- **Retrofit** 2.11.0 - HTTP клиент
- **Dagger Hilt** 2.52 - внедрение зависимостей
- **Navigation Component** 2.8.4 - навигация между экранами
- **ViewBinding** - привязка представлений
- **Glide** 4.16.0 - загрузка изображений
- **Google Maps** 19.0.0 - интеграция с картами

### Дополнительные инструменты
- **OkHttp** 4.12.0 - HTTP клиент
- **Gson** - сериализация JSON
- **Material Design** - UI компоненты
- **Desugaring** - поддержка java.time на старых версиях Android

## Установка и запуск

### Требования
- Android Studio Arctic Fox или новее
- Android SDK 24+ (Android 7.0)
- Kotlin 1.9.24+
- Gradle 8.11.1+

### Настройка
1. Клонируйте репозиторий
2. Откройте проект в Android Studio
3. Создайте файл `app/src/main/res/values/api_key.xml`:
   ```xml
   <?xml version="1.0" encoding="utf-8"?>
   <resources>
       <string name="api_key">YOUR_API_KEY_HERE</string>
       <string name="base_url">http://94.228.125.136:8080/</string>
   </resources>
   ```
4. Создайте файл `app/src/main/res/values/google_maps_api.xml`:
   ```xml
   <?xml version="1.0" encoding="utf-8"?>
   <resources>
       <string name="google_maps_key">YOUR_GOOGLE_MAPS_API_KEY_HERE</string>
   </resources>
   ```
5. Соберите и запустите проект

## API

Приложение использует REST API для взаимодействия с сервером:
- **Base URL**: `http://94.228.125.136:8080/`
- **Авторизация**: Bearer Token
- **API Key**: передается в заголовке `Api-Key`

### Основные endpoints:
- `POST /users/authentication` - авторизация
- `POST /users/registration` - регистрация
- `GET /posts` - получение постов
- `POST /posts` - создание поста
- `GET /events` - получение событий
- `POST /events` - создание события
- `POST /media` - загрузка медиафайлов

## Особенности реализации

### Архитектурные решения
- **Single Activity** - упрощает навигацию и управление состоянием
- **Repository Pattern** - изолирует источники данных
- **MVVM** - четкое разделение ответственности
- **Dependency Injection** - упрощает тестирование и поддержку

### Асинхронность
- **Kotlin Coroutines** для всех сетевых операций
- **StateFlow** для реактивного состояния
- **LiveData** для UI обновлений

### Безопасность
- API ключи не коммитятся в репозиторий
- Валидация пользовательского ввода
- Обработка ошибок сети

### UX/UI
- **Material Design** компоненты
- Адаптивный дизайн
- Интуитивная навигация
- Обратная связь пользователю (Toast, Progress)
